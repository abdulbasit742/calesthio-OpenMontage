# Archive Register Lookup Note

This note explains how a future reviewer can locate the archived package by using the register ID.

## Purpose

The lookup note turns the register ID into a simple search path for the archive package.

Use it after the package index has been updated with both the storage receipt ID and the register ID.

## Lookup Inputs

A reviewer should have these values before lookup:

- package label
- storage receipt ID
- register ID
- storage owner
- storage location
- package index reference

## Lookup Steps

1. Open the archive package index.
2. Search for the register ID.
3. Confirm the receipt ID beside the register ID.
4. Confirm the storage owner and storage location.
5. Open the storage receipt report.
6. Confirm the receipt status is `received`.
7. Use the storage location to find the stored archive package.

## Match Rule

The lookup is valid when the register ID, receipt ID, storage owner, and storage location match across the package index and the storage receipt records.

## Reviewer Note

Keep this lookup note with the index note and index checks so a future reviewer can find the package without rebuilding the closeout workflow.
